# ネット競馬のレース検索のページから中央競馬の全レース施行条件について過去５年分の競争をサンプルとして、それぞれについて平均タイムをだすプログラムです。
# 平均の集計期間については容易に変更が可能です。
# 下記具体例の　&start_year=2019&start_mon=1&end_year=2023&end_mon=12　の部分を書き換えることで集計期間をコントロールできます。
# 直上では２０１９年１月頭～２０２３年１２月末までの計算５年間を集計期間としています。
# （割とそのままURLになっているので書き換え方は直感的にわかると思います。）
#
# 取得するページのURLの規則性は以下の通りです。（この規則性を利用して多重forループを組んで全レース施行条件についてスクレイピングを行うプログラムとなっている）
# (法則性を理解することで各種条件についても取得範囲のコントロールが可能です。プログラム中のリストから対応要素を消すと狙ったものだけを大体取得できます。）

# 具体例：https://db.netkeiba.com/?pid=race_list&word=&track%5B%5D=1&start_year=2019&start_mon=1&end_year=2023&end_mon=12&jyo%5B%5D=01&grade%5B%5D=5&kyori_min=&kyori_max=&kyori%5B%5D=1200&sort=date&list=100
# の場合

# https://db.netkeiba.com/?pid=race_list&word= ここは共通　&sort=date&list=100 の部分も共通です。&kyori_min=&kyori_max=　も今回の目的では指定しないので共通。

# URL全体は以下のような規則性でURLにて要素が並んでいる
# 芝かダートか障害競走か　⇒　調査期間　⇒　競馬場　⇒　（距離調査区間幅：&kyori_min=&kyori_max=：使わない＆共通）⇒距離　⇒　ページが何枚目かの指定（ない場合もある）

# &track%5B%5D=1 左の部分の末尾が　1だと芝　2だとダート　3だと障害　となっている。

# 競馬場の番号の法則性はいつも通り　01が札幌　02が函館　03が福島　04が新潟　05が東京　06が中山　07が中京　08が京都　09が阪神　10が小倉　となっている。
# &jyo%5B%5D=01& なら　札幌競馬場の情報を取得

# グレード(取得するレースのクラス）の法則性　&grade%5B%5D=11　のように末尾の数字がグレードに対応している
# 1がG1　2がG2　3がG3　11がL　4がOP　5が3勝クラス 6が2勝クラス 7が1勝クラス 8が新馬戦　9が未勝利　（10が未出走：使わない）

# &kyori%5B%5D=1200　これはそのまま末尾が距離に対応している（この場合1200ｍの競争を指定）
